## Overview

Aegis provides a comprehensive set of block templates that define the layout structure for different page types. Templates are HTML files in the `templates/` directory that use block markup.

## Available Templates

### Core Templates

| **Index** | `index.html` | Default fallback template for all content |
| **Front Page** | `front-page.html` | Homepage template |

| **Single** | `single.html` | Single post template |
| **Archive** | `archive.html` | Post archive/listing template |

| **404** | `404.html` | Error page template |

## Template Structure

Each template is an HTML file containing block markup. A typical template includes:

```html
<!-- wp:template-part {"slug":"header","area":"header"} /-->

<!-- wp:group {"tagName":"main","layout":{"type":"constrained"}} -->
<main class="wp-block-group">
  <!-- wp:post-content /-->
</main>
<!-- /wp:group -->

<!-- wp:template-part {"slug":"footer","area":"footer"} /-->
```

## Template Hierarchy

Aegis follows the standard WordPress template hierarchy:

```
Specific → General → Fallback

single-{post-type}-{slug}.html
  → single-{post-type}.html
    → single.html
      → singular.html
        → index.html
```

## Assigning Templates

### Per-Page Assignment

<Steps>
  <Step title="Edit page">
    Edit any page or post.
  </Step>
  <Step title="Open settings">
    Open the **Settings** sidebar (gear icon).
  </Step>
  <Step title="Select template">
    Under **Template**, select from available templates.
  </Step>
</Steps>

Templates marked with specific post types only appear for those types.

### Default Templates

- **Pages** use `page.html` by default
- **Posts** use `single.html` by default
- **Archives** use `archive.html` by default
- **Homepage** uses `front-page.html` if set, otherwise `index.html`

## Editing Templates

### Via Site Editor

1. Go to **Appearance → Editor → Templates**.
2. Click any template to edit it.
3. Modifications are saved to the database, overriding the theme file.
4. Click **Reset** to revert to the theme's original template.

### Via Theme Files

Edit the HTML files directly in `templates/`. Changes take effect immediately unless a database override exists.

## Custom Template Registration

Custom templates are registered in `theme.json` under `customTemplates`:

```json
{
  "customTemplates": [
    {
      "title": "Full Width",
      "name": "full-width",
      "postTypes": ["page", "post"]
    }
  ]
}
```

- **`title`** — Display name in the template selector
- **`name`** — Must match the filename (without `.html`)
- **`postTypes`** — Optional array restricting which post types can use this template
