## Overview

Aegis Docs is a companion plugin that adds a full-featured documentation system to your WordPress site, built on the block editor and Aegis design system.

## Requirements

- WordPress 6.4+
- Aegis theme (active)
- PHP 8.1+

## Features

### Custom Post Type

The plugin registers an `aegis_doc` custom post type for documentation pages:

- Hierarchical (supports parent/child pages)
- Block editor enabled
- Supports title, editor, excerpt, thumbnail, page attributes, and custom fields
- REST API enabled

### Documentation Spaces

A custom taxonomy `doc_space` organizes documentation into separate spaces (e.g., "User Guide", "API Reference", "Developer Docs"):

- Hierarchical taxonomy
- Each space can have its own navigation tree
- Spaces are filterable in the REST API

### Block Patterns

The plugin includes 10+ documentation-specific patterns:

| Pattern | Description |
|---------|-------------|
| **Hero Docs** | Documentation landing page hero section |
| **Quick Links** | Grid of quick navigation links |
| **Card Grid** | Documentation category cards |
| **Callout** | Info, warning, success, and danger callout boxes |
| **Steps** | Numbered step-by-step guide with timeline |
| **Code Group** | Tabbed multi-language code blocks |
| **Card Link** | Single navigation card with icon |
| **Feature Grid** | 3-column feature showcase |
| **API Endpoint** | REST API endpoint documentation with method badge |
| **Pro Badge** | Inline badge marking Pro-only features |

### Templates

| Template | Description |
|----------|-------------|
| **Docs Archive** | Documentation listing/index page |
| **Single Doc** | Individual documentation page with sidebar |

### Template Parts

| Part | Description |
|------|-------------|
| **Docs Header** | Documentation-specific header with search |
| **Docs Sidebar** | Navigation sidebar with doc tree |
| **Docs Footer** | Documentation footer with links |

### REST API

The plugin exposes documentation data via the WordPress REST API under the `aegis-docs/v1` namespace:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/docs` | GET | List all documentation pages |
| `/docs/{id}` | GET | Get a single documentation page |
| `/docs/{id}/children` | GET | Get child pages of a document |
| `/spaces` | GET | List all documentation spaces |
| `/spaces/{slug}/docs` | GET | Get documents in a specific space |
| `/search` | GET | Search documentation content |

#### Search Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `s` | string | Search query |
| `space` | string | Filter by space slug |
| `per_page` | integer | Results per page (default: 10) |
| `page` | integer | Page number |

### CSS Assets

- **Frontend CSS** — Layout, sidebar, TOC, breadcrumbs, navigation, patterns
- **Editor CSS** — Block preview styles matching frontend appearance

## Installation

<Steps>
  <Step title="Activate theme">
    Ensure the Aegis theme is active.
  </Step>
  <Step title="Upload plugin">
    Upload the `aegis-docs` plugin to `wp-content/plugins/`.
  </Step>
  <Step title="Activate">
    Activate via **Plugins → Installed Plugins**.
  </Step>
  <Step title="Create spaces">
    Create documentation spaces under **Docs → Spaces**.
  </Step>
  <Step title="Add content">
    Start creating documentation pages under **Docs → Add New**.
  </Step>
</Steps>

## Configuration

The plugin works out of the box with sensible defaults. Documentation pages automatically use the docs templates and sidebar navigation.

## Customization

### Templates

Override plugin templates by creating matching files in your theme's `templates/` directory.

### Patterns

Plugin patterns can be customized after insertion — all blocks within patterns are fully editable.

### Styles

Plugin styles use Aegis design tokens and can be overridden via the Site Editor or custom CSS.
