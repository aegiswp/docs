## Overview

All colors in Aegis are available as CSS custom properties generated from `theme.json`.

## Usage Examples

### CSS

```css
.my-element {
    background-color: var(--wp--preset--color--neutral-100);
    color: var(--wp--preset--color--neutral-800);
    border-color: var(--wp--preset--color--neutral-200);
}

.my-button {
    background-color: var(--wp--preset--color--primary-500);
    color: var(--wp--preset--color--primary-25);
}

.success-message {
    background-color: var(--wp--preset--color--success-100);
    color: var(--wp--preset--color--success-600);
}
```

### Block Editor

In block attributes:

```json
{
    "backgroundColor": "primary-500",
    "textColor": "neutral-0"
}
```

### theme.json

```json
{
    "styles": {
        "color": {
            "background": "var(--wp--preset--color--neutral-0)",
            "text": "var(--wp--preset--color--neutral-600)"
        }
    }
}
```

## Color Relationships

### Text on Backgrounds

| Background | Recommended Text |
|------------|------------------|
| `neutral-0` | `neutral-600`, `neutral-800` |
| `neutral-100` | `neutral-700`, `neutral-800` |
| `primary-500` | `primary-25`, `neutral-0` |
| `neutral-950` | `neutral-100`, `neutral-0` |

### Borders

| Context | Recommended Border |
|---------|-------------------|
| Light backgrounds | `neutral-200` |
| Dark backgrounds | `neutral-700` |
| Focus states | `primary-500` |

<Tip>
All color combinations in Aegis meet WCAG 2.1 AA contrast requirements.
</Tip>
