## Overview

Aegis Pro extends the core Query Loop block (`core/query`) with 19 feature classes providing AJAX pagination, carousel and masonry layouts, frontend filters, dynamic image sources, block bindings, conditional visibility, and performance optimizations. All settings appear in the block sidebar when editing a Query Loop block.

## AJAX Pagination

Replace standard page reloads with seamless AJAX-powered pagination.

### Pagination Types

| Type | Description |
|------|-------------|
| **Load More** | Button appends the next page of results |
| **Infinite Scroll** | Automatically loads more as the user scrolls |
| **AJAX Pages** | Replaces pagination links with AJAX requests |

### Attributes

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `aegisProAjaxPagination` | boolean | `false` | Enable AJAX pagination |
| `aegisProPaginationType` | string | `load_more` | Type: `load_more`, `infinite_scroll`, `ajax_pages` |
| `aegisProLoadMoreText` | string | `Load More` | Button label |
| `aegisProLoadingText` | string | `Loading...` | Loading state text |
| `aegisProNoMoreText` | string | `No more posts` | End-of-results text |
| `aegisProScrollThreshold` | number | `300` | Infinite scroll trigger distance (px from bottom) |
| `aegisProLoadAnimation` | string | `fade` | Animation: `fade`, `slide`, `none` |

### REST API

`POST /aegis-pro/v1/query/paginate` — Fetches the next page of query results server-side and returns rendered HTML.

**Class:** `AegisPro\Query\AjaxPagination`

| `aegisProCarouselNavPosition` | string | `sides` | Position: `sides`, `bottom`, `outside` |
| `aegisProCarouselPagination` | boolean | `true` | Show pagination |

| `aegisProCarouselAutoplay` | boolean | `false` | Enable autoplay |

## Image Source

### Source Types

| Source | Description |
|--------|-------------|
| **Featured** | Default WordPress featured image |
| **Content** | Nth image from post content (by index) |
| **ACF** | Image from an ACF image field |
| **Meta Box** | Image from a Meta Box field |

### Attributes

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `aegisProImageSource` | string | `featured` | Source: `featured`, `content`, `acf`, `metabox` |
| `aegisProImageIndex` | number | `1` | Content image index (1st, 2nd, etc.) |
| `aegisProImageFallback` | string | `featured` | Fallback if primary source is empty |

**Class:** `AegisPro\Query\ImageSource`

## Block Bindings

WordPress Block Bindings API integration (requires WP 6.5+) for dynamic data display without custom templates.

### Binding Sources

| Source Name | Description |
|-------------|-------------|
| `aegis-pro/query-data` | Bind to post meta and computed values |
| `aegis-pro/acf-field` | Bind to ACF field values |
| `aegis-pro/metabox-field` | Bind to Meta Box field values |

### Usage

In the block editor, select a block inside a Query Loop and use the Bindings panel to connect block attributes to dynamic data sources.

```html
<!-- wp:paragraph {"metadata":{"bindings":{"content":{"source":"aegis-pro/query-data","args":{"key":"custom_field_name"}}}}} -->
<p></p>
<!-- /wp:paragraph -->
```

**Class:** `AegisPro\Query\BlockBindings`

## Block Conditions

## Query Performance

Caching and optimization features for Query Loop blocks.

### Features

- **Transient caching** — Cache rendered query results with configurable duration and automatic invalidation on post save
- **Lazy loading** — Defer image loading with configurable viewport threshold
- **Image preloading** — Preload the first N images for above-the-fold performance
- **Query optimization** — Disable `SQL_CALC_FOUND_ROWS` for queries that don't need total counts
- **Meta query optimization** — Optimize complex meta queries
- **Skeleton loading** — Show placeholder skeleton UI while content loads

### Attributes

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `aegisProCacheEnabled` | boolean | `false` | Enable transient caching |
| `aegisProCacheDuration` | number | `3600` | Cache TTL in seconds (default 1 hour) |
| `aegisProLazyLoadImages` | boolean | `true` | Lazy load images |
| `aegisProLazyLoadThreshold` | number | `200` | Lazy load trigger distance (px) |
| `aegisProPreloadCount` | number | `3` | Preload first N images |
| `aegisProNoFoundRows` | boolean | `false` | Disable `SQL_CALC_FOUND_ROWS` |
| `aegisProOptimizeMeta` | boolean | `true` | Optimize meta queries |
| `aegisProSkeletonLoading` | boolean | `false` | Show skeleton placeholders |

**Class:** `AegisPro\Query\QueryPerformance`

## Architecture

All Query Pro classes follow the same pattern:

1. **Attribute registration** — Hook `register_block_type_args` to add custom attributes to `core/query` (or `core/post-featured-image` for ImageSource)
2. **Render modification** — Hook `render_block_core/query` to wrap or modify the block output
3. **Conditional loading** — Assets (JS/CSS) are only enqueued when the feature is enabled on the current page
4. **Interactivity API** — Client-side behavior uses `wp_interactivity_state()` for reactive updates

### Hook Priorities

| Priority | Class | Hook |
|----------|-------|------|
| 10 | AjaxPagination | `render_block_core/query` |
| 11 | FrontendFilters | `render_block_core/query` |
| 12 | BlockConditions | `render_block` |
| 13 | CarouselLayout | `render_block_core/query` |
| 15 | ImageSource | `render_block_core/post-featured-image` |
