## Overview

Aegis is designed with accessibility at its core, prioritizing WCAG compliance, semantic HTML, and inclusive design patterns.

## Core Principles

### Semantic HTML

All blocks and patterns use proper semantic HTML elements:

- `<header>`, `<main>`, `<footer>`, `<nav>` for page structure
- `<article>` for post content
- `<figure>` and `<figcaption>` for media
- `<blockquote>` for quotes
- Proper heading hierarchy (H1 → H6)

### Color Contrast

Aegis color tokens are optimized for WCAG AA contrast compliance:

- **Body text** (`neutral-600`) on **background** (`neutral-0`) — passes AA
- **Heading text** (`neutral-800`) on **background** (`neutral-0`) — passes AAA
- **Button text** (`primary-25`) on **button background** (`primary-500`) — passes AA
- Semantic colors (success, warning, error) maintain contrast in both light and dark modes

### Color Vision Deficiency

The theme is specifically optimized for users with color vision deficiency:

- Color is never the sole indicator of meaning
- Semantic colors include sufficient luminance contrast
- Icons and text labels supplement color-coded information

## Keyboard Navigation

- **Focus indicators** — All interactive elements have visible focus styles
- **Tab order** — Logical tab order follows visual layout
- **Skip links** — Skip to main content link for keyboard users
- **Escape key** — Closes modals, search overlays, and menus
- **Arrow keys** — Navigate within menus and sliders

## ARIA Support

### Modal Block

- `role="dialog"` and `aria-modal="true"`
- Focus trapping within the modal
- `aria-labelledby` pointing to the modal title
- Escape key to close

### Slider Block

- `role="region"` with `aria-label`
- Slide navigation announced to screen readers
- Pause/play controls for autoplay
- `aria-live` region for slide changes

### Navigation

- `aria-expanded` for submenu toggles
- `aria-current="page"` for active menu items
- Proper `role` attributes for mega menu structures

### Accordion List

- `aria-expanded` for toggle state
- `aria-controls` linking trigger to content
- Keyboard toggle via Enter/Space

## Reduced Motion

Aegis respects the `prefers-reduced-motion` media query:

- Animations and transitions are disabled or minimized
- Slider autoplay is paused
- Marquee scrolling stops
- Counter animations complete instantly

## Form Accessibility

Form elements (inputs, selects, textareas) include:

- Visible labels
- Focus styles with `primary-500` border color
- Error states using semantic error colors
- Sufficient touch target sizes

## Testing Accessibility

### Automated Tools

- **Lighthouse** — Accessibility audit in Chrome DevTools
- **axe DevTools** — Browser extension for WCAG testing
- **WAVE** — Web accessibility evaluation tool

### Manual Testing

1. **Keyboard-only navigation** — Tab through the entire page
2. **Screen reader** — Test with NVDA (Windows), VoiceOver (macOS), or JAWS
3. **Zoom** — Test at 200% and 400% zoom levels
4. **High contrast mode** — Test with Windows High Contrast Mode
5. **Color blindness simulation** — Use Chrome DevTools rendering emulation

## Best Practices for Content Authors

1. **Use proper heading hierarchy** — Don't skip heading levels
2. **Add alt text** — Describe all meaningful images
3. **Use descriptive link text** — Avoid "click here" or "read more"
4. **Provide captions** — Add captions to videos and complex media
5. **Use sufficient contrast** — Stick to Aegis color tokens
6. **Test with keyboard** — Ensure all interactive elements are reachable
