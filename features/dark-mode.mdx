## Overview

Dark mode in Aegis is:

- **Automatic** — Detects `prefers-color-scheme: dark` from the user's OS
- **Zero JavaScript** — Default behavior requires no JS toggle
- **Token-based** — All color tokens remap automatically via shade inversion
- **FOUC-free** — No flash of light mode on page load
- **Cookie-aware** — Respects user preference stored in `aegisDarkMode` cookie
- **URL-overridable** — `?dark_mode=true` or `?dark_mode=false` query parameter

## How It Works

The framework's `DarkMode` class (`DesignSystem/DarkMode.php`) implements the `Styleable` interface and generates CSS that remaps every color token shade to its dark-mode counterpart inside a `@media (prefers-color-scheme: dark)` block.

### Token Remapping

Each color scale is inverted so light shades become dark and vice versa:

| Light Shade | → Dark Shade | Scale |
|-------------|-------------|-------|
| `primary-950` | → `primary-50` | Primary, Secondary |
| `primary-25` | → `primary-1000` | Primary, Secondary |
| `neutral-950` | → `neutral-0` | Neutral |
| `neutral-0` | → `neutral-950` | Neutral |
| `success-600` | → `success-100` | Success, Warning, Error |
| `success-100` | → `success-600` | Success, Warning, Error |

The `500` shade stays at `500` for all scales (the midpoint is stable).

### Body Classes

The framework adds body classes based on the user's preference:

| Class | Meaning |
|-------|---------|
| `default-mode-light` | Global default is light |
| `default-mode-dark` | Global default is dark |
| `default-mode-system` | Follow OS preference |
| `is-style-dark` | Dark mode forced on (cookie or URL) |
| `is-style-light` | Light mode forced on (cookie or URL) |

Priority: URL parameter > Cookie > Global setting > OS preference.

## Configuration

`User`

Dark mode settings are available in the page editor sidebar under Aegis settings:

| Setting | Description |
|---------|-------------|
| **Enable/Disable** | Toggle dark mode support globally via `theme.json` → `custom.darkMode` |
| **Per-Page Override** | Disable dark mode on specific pages via post meta |
| **Custom Colors** | Override specific dark mode color mappings |

### Per-Page Override

In the page/post editor sidebar, toggle "Disable Dark Mode" to prevent dark mode on that specific page. This removes the dark mode CSS output for that page only.

## Best Practices

1. **Always use Aegis color tokens** — Hardcoded hex values won't adapt to dark mode
2. **Test both modes** — Use OS settings or browser DevTools to toggle `prefers-color-scheme`
3. **Check contrast** — Ensure text remains readable in both modes
4. **Consider images** — Some images may need alternate versions or CSS filters for dark backgrounds
5. **Use semantic tokens** — Prefer `neutral-*` over specific hex values for backgrounds and text

## For Developers

`Dev`

### Detect Dark Mode in CSS

```css
@media (prefers-color-scheme: dark) {
  .my-custom-element {
    background: var(--wp--preset--color--neutral-900);
  }
}
```

Or target the body class for cookie/URL-forced dark mode:

```css
.is-style-dark .my-custom-element {
  background: var(--wp--preset--color--neutral-900);
}
```

### Detect Dark Mode in JavaScript

```js
const isDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
const isForcedDark = document.body.classList.contains('is-style-dark');
```

### Filter the Shade Map

Use the `aegis_dark_mode_map` filter to customize which shades remap:

```php
add_filter('aegis_dark_mode_map', function(array $map): array {
    // Keep primary-500 as-is in dark mode
    $map['primary'][500] = 500;
    return $map;
});
```

### Implementation

- **Class:** `Aegis\Framework\DesignSystem\DarkMode`
- **Interface:** `Styleable` — generates CSS via the inline assets system
- **Dependencies:** `CustomProperties`, `Color` utility, `JSON` utility
- **Cookie:** `aegisDarkMode` — values: `true`, `false`, `system`
- **URL param:** `?dark_mode=true` or `?dark_mode=false`
