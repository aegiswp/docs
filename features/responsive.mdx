## Overview

Aegis includes built-in responsive utilities that ensure your designs look perfect on all devices.

## Reverse on Mobile

Reverse the order of columns on mobile devices:

- Available on the **Columns** block
- Stacked columns appear in reverse order on small screens
- Useful for alternating text/image layouts that should maintain visual flow on mobile

## Hide on Mobile

Control block visibility at different breakpoints:

- Hide specific blocks on mobile devices
- Show mobile-only content that's hidden on desktop
- Applied via block-level CSS classes

## Responsive Spacer

The **Spacer** block is enhanced with responsive breakpoint controls:

- Set different heights for mobile, tablet, and desktop
- Prevents excessive whitespace on small screens
- Maintains proportional spacing across devices

## Fluid Typography

All font sizes use `clamp()` for smooth scaling between breakpoints:

```css
/* Example: 44px heading */
font-size: clamp(40px, 4.4vw, 44px);
```

This means typography scales fluidly with the viewport width, eliminating jarring size jumps at breakpoints.

## Fluid Spacing

Spacing tokens reference the fluid font size scale, so spacing also scales responsively:

```css
/* Large spacing = 48px font size token */
--wp--preset--spacing--lg: var(--wp--preset--font-size--48);
/* Which resolves to: clamp(40px, 4.8vw, 48px) */
```

## Layout Constraints

Content and wide widths use responsive calculations:

```css
/* Content width: max 720px, with responsive padding */
min(calc(100dvw - var(--wp--preset--spacing--lg) * 2), 720px)

/* Wide width: max 1620px, with responsive padding */
min(calc(100dvw - var(--wp--preset--spacing--lg) * 2), 1620px)
```

## Root Padding

The `useRootPaddingAwareAlignments` setting ensures:

- Body padding of `var(--wp--preset--spacing--lg)` (48px) on left and right
- Full-width blocks break out of this padding automatically
- Content-width blocks respect the padding

## Mobile Navigation

The Navigation block automatically converts to a hamburger menu on small screens. See [Navigation](/features/navigation) for details.

## Best Practices

1. **Use preset tokens** — They're already responsive via `clamp()`
2. **Test at multiple widths** — Use the Site Editor's responsive preview
3. **Avoid fixed pixel values** — Use relative units or preset tokens
4. **Consider touch targets** — Minimum 44×44px for interactive elements
5. **Stack columns** — Columns automatically stack on mobile by default
