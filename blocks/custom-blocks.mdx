## Overview

Aegis includes custom blocks built with the WordPress Interactivity API for reactive, JavaScript-powered UI components. These blocks require a build step and are registered via `block.json` metadata.

| Block | Name | Tier | Description |
|-------|------|------|-------------|
| Countdown | `aegis/countdown` | `Free` `Pro` | Countdown timer with customizable segments |
| Modal | `aegis/modal` | `Free` `Pro` | Accessible modal dialogs |
| Slider | `aegis/slider` | `Free` `Pro` | Responsive carousels |
| Slide | `aegis/slide` | `Free` | Individual slides within sliders |
| Map | `aegis/map` | `Free` `Pro` | Interactive Google Maps |
| Related Posts | `aegis/related-posts` | `Free` | Taxonomy-based related content |
| Toggle | `aegis/toggle` | `Free` `Pro` | Accessible accordion/toggle sections |
| Toggle Content | `aegis/toggle-content` | `Free` | Content area within toggles |
| Video | `aegis/video` | `Free` `Pro` | Enhanced video player with custom controls |

## Countdown Block

`Free` `Pro`

A countdown timer that counts down to a specific date and time with customizable segments, labels, separators, and expiry behavior.

### Features

- **Configurable segments** — Show or hide days, hours, minutes, seconds independently
- **Custom labels** — Override default segment labels
- **Separator styles** — Colon, dot, dash, or none
- **Layout modes** — Inline or stacked
- **Expiry message** — Display custom text when the countdown reaches zero
- **Timezone** — UTC or visitor's local time
- **Server-side rendered** — PHP render callback with `view.js` for client-side ticking

### Attributes

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `datetime` | string | `""` | Target date/time (ISO 8601) |
| `showDays` | boolean | `true` | Show days segment |
| `showHours` | boolean | `true` | Show hours segment |
| `showMinutes` | boolean | `true` | Show minutes segment |
| `showSeconds` | boolean | `true` | Show seconds segment |
| `labels` | object | `{days, hours, minutes, seconds}` | Custom segment labels |
| `separator` | string | `colon` | Separator style: `colon`, `dot`, `dash`, `none` |
| `layout` | string | `inline` | Layout: `inline` or `stacked` |
| `expiryMessage` | string | `""` | Message shown when countdown ends |
| `timezone` | string | `utc` | Timezone: `utc` or `local` |

### Usage

Search for "Countdown" in the block inserter. Set a target date/time, choose which segments to display, and optionally configure an expiry message.

### Pro Extensions

<Tip>
[Aegis Pro](/plugins/aegis-pro) adds evergreen timers (per-visitor countdowns), animation styles, auto-restart, expiry actions (redirect, hide, replace), urgency styling, and Event schema markup.
</Tip>

## Modal Block

An accessible modal dialog with multiple display modes.

### Features

- **Popup mode** - Centered overlay dialog
- **Off-canvas mode** - Slide-in from edge
- **Bottom sheet mode** - Mobile-friendly bottom panel
- **Fullscreen mode** - Full viewport coverage

### Usage

```html
<!-- wp:aegis/modal {"mode":"popup"} -->
<div class="wp-block-aegis-modal">
  <!-- Trigger button -->
  <!-- Modal content -->
</div>
<!-- /wp:aegis/modal -->
```

### Settings

| Setting | Options | Description |
|---------|---------|-------------|
| Mode | popup, off-canvas, bottom-sheet, fullscreen | Display style |
| Trigger | button, link, auto | How modal opens |
| Close on overlay | true/false | Click outside to close |
| Close on escape | true/false | ESC key closes modal |

### Accessibility

The Modal block is built with accessibility in mind:

- Focus trapping within modal
- ESC key to close
- ARIA attributes for screen readers
- Focus returns to trigger on close

## Slider Block

A responsive slider/carousel powered by Splide.js with extensive configuration options.

### Features

- Multiple slide types (content, image, video)
- Configurable slides per page and per move
- Autoplay with pause on hover
- Loop and drag support
- Arrow and dot navigation
- Customizable speed and interval
- RTL direction support
- Custom height settings
- Responsive breakpoints

### Attributes

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `type` | string | `slide` | Slider type |
| `perPage` | number | `1` | Slides visible at once |
| `perMove` | number | `1` | Slides to move per interaction |
| `autoplay` | boolean | `false` | Auto-advance slides |
| `pauseOnHover` | boolean | `true` | Pause autoplay on hover |
| `loop` | boolean | `false` | Infinite loop |
| `drag` | boolean | `true` | Enable drag/swipe |
| `showArrows` | boolean | `true` | Show navigation arrows |
| `showDots` | boolean | `true` | Show pagination dots |
| `speed` | number | `400` | Transition speed (ms) |
| `interval` | number | `5000` | Autoplay interval (ms) |
| `direction` | string | `ltr` | Slide direction |
| `height` | string | — | Custom slider height |
| `breakpoints` | object | — | Responsive breakpoint config |

### Usage

Search for "Slider" in the block inserter. Add Slide blocks inside the Slider, then configure options in the block sidebar.

## Slide Block

An individual slide within a Slider block.

### Features

- Must be a child of `aegis/slider`
- Supports any block content
- Color and spacing controls
- Custom class name support

### Usage

Slide blocks are automatically available when editing inside a Slider block. Add content blocks inside each Slide.

<Note>
The Slide block can only be used inside a Slider block. It won't appear in the inserter outside of a slider context.
</Note>

## Map Block

An interactive Google Maps block with a performance-optimized facade pattern.

### Features

- Google Maps embed with configurable center, zoom, and map type
- Multiple marker support with info windows
- Facade pattern — loads a static image placeholder until user interaction, then initializes the full map
- Responsive sizing
- Keyboard accessible

### Attributes

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `lat` | number | `40.7128` | Map center latitude |
| `lng` | number | `-74.006` | Map center longitude |
| `zoom` | number | `12` | Zoom level (1–20) |
| `mapType` | string | `roadmap` | Map type (`roadmap`, `satellite`, `hybrid`, `terrain`) |
| `markers` | array | `[]` | Array of marker objects with lat, lng, title |
| `height` | string | `400px` | Map height |

### Usage

Search for "Map" in the block inserter. Set your Google Maps API key in the Aegis admin settings, then configure the map center, zoom, and markers in the block sidebar.

### Pro Extensions

<Tip>
[Aegis Pro](/plugins/aegis-pro) adds custom markers, clustering, directions, store locator, geolocation, heatmap, drawing tools, data import, Schema.org output, and dynamic markers.
</Tip>

## Related Posts Block

`Free`

Displays posts related to the current content by shared taxonomy terms, with multiple layout variants and fallback options.

### Features

- **Taxonomy-based matching** — Auto-detect, category, or tag-based relation
- **4 style variants** — Grid, list, cards, minimal
- **Configurable elements** — Featured image, date, excerpt, category badge
- **Fallback behavior** — Show latest posts or hide when no related posts found
- **Image aspect ratios** — 16/9, 4/3, 1/1, 3/2
- **Server-side rendered** — PHP render callback for SEO-friendly output

### Attributes

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `postsPerPage` | number | `3` | Number of related posts |
| `columns` | number | `3` | Grid columns |
| `showFeaturedImage` | boolean | `true` | Show featured image |
| `showDate` | boolean | `true` | Show publish date |
| `showExcerpt` | boolean | `true` | Show excerpt |
| `showCategory` | boolean | `true` | Show category badge |
| `heading` | string | `"Related Posts"` | Section heading text |
| `headingTag` | string | `h2` | Heading element: `h2`–`h6` |
| `styleVariant` | string | `grid` | Layout: `grid`, `list`, `cards`, `minimal` |
| `taxonomySource` | string | `auto` | Relation source: `auto`, `category`, `post_tag` |
| `fallbackBehavior` | string | `latest` | Fallback: `latest` or `hide` |
| `excerptLength` | number | `20` | Excerpt word count |
| `imageAspectRatio` | string | `16/9` | Image ratio: `16/9`, `4/3`, `1/1`, `3/2` |

### Usage

Search for "Related Posts" in the block inserter. Place it in your single post template. The block automatically queries posts sharing taxonomy terms with the current post.

## Toggle Block

`Free` `Pro`

An accessible accordion/toggle block with expandable content sections, ideal for FAQs, collapsible content, and progressive disclosure.

### Features

- **Accessible** — Built with proper ARIA attributes and keyboard support
- **Configurable heading** — Choose heading level (H2–H6 or paragraph)
- **Icon options** — Chevron, plus, or arrow icons, positioned left or right
- **Animation** — Configurable expand/collapse duration
- **Multiple open** — Allow multiple toggles open simultaneously or single-open accordion mode
- **Default state** — Start open or closed

### Attributes

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `heading` | string | `""` | Toggle heading text |
| `headingTag` | string | `h3` | Heading element: `h2`–`h6`, `p` |
| `isOpen` | boolean | `false` | Start expanded |
| `iconPosition` | string | `right` | Icon position: `left` or `right` |
| `iconType` | string | `chevron` | Icon style: `chevron`, `plus`, `arrow` |
| `allowMultiple` | boolean | `true` | Allow multiple toggles open at once |
| `animationDuration` | number | `300` | Animation duration in ms |

### Usage

Search for "Toggle" in the block inserter. Add a heading, then place any blocks inside the Toggle Content area. Group multiple toggles for an FAQ section.

```html
<!-- wp:aegis/toggle {"heading":"How does it work?"} -->
  <!-- wp:aegis/toggle-content -->
    <!-- wp:paragraph -->
    <p>Your answer content here.</p>
    <!-- /wp:paragraph -->
  <!-- /wp:aegis/toggle-content -->
<!-- /wp:aegis/toggle -->
```

### Toggle Content Block

The `aegis/toggle-content` block is a child block that can only be used inside `aegis/toggle`. It serves as the expandable content area and supports any inner blocks.

### Pro Extensions

<Tip>
[Aegis Pro](/plugins/aegis-pro) extends the Toggle block with additional animation styles, conditional display rules, and advanced configuration options.
</Tip>

## Best Practices

### Modal

- Use sparingly—modals interrupt user flow
- Keep content focused and concise
- Provide clear close mechanism
- Test keyboard navigation

### Slider

- Limit to 5-7 slides maximum
- Ensure content is readable at all sizes
- Provide navigation controls
- Consider auto-play carefully (can be annoying)
- Test touch interactions on mobile

### Countdown

- Set realistic deadlines — avoid countdowns to dates far in the future
- Always provide an expiry message so users see meaningful content after the timer ends
- Use the stacked layout for mobile-heavy audiences
- Consider timezone differences — use `local` for visitor-specific events

### Toggle

- Use descriptive headings — users should understand the content before expanding
- Keep toggle content concise
- Use single-open accordion mode for long FAQ lists to reduce scrolling
- Ensure heading levels follow the page hierarchy

### Performance

- Optimize images within sliders and related posts
- Use lazy loading for off-screen slides
- Minimize JavaScript in modal content
- Test on slower devices

<Tip>
For testimonials or simple image galleries, consider using patterns with the Query Loop block instead of sliders for better performance and accessibility.
</Tip>

## Technical Details

### Registration

Custom blocks are registered via PHP in the framework's `CoreBlocks` namespace:

```php
// Slider.php registers both aegis/slider and aegis/slide
register_block_type( get_template_directory() . '/src/Blocks/slider' );
register_block_type( get_template_directory() . '/src/Blocks/slide' );
```

Each block has a `block.json` metadata file in `src/Blocks/{block-name}/`.

### Interactivity API

Custom blocks use the WordPress Interactivity API for reactive behavior:

- **Server-side rendering** — PHP render callbacks generate initial HTML with `data-wp-*` directives.
- **Client-side hydration** — JavaScript modules hydrate the server-rendered HTML for interactivity.
- **State management** — Shared state via `wp_interactivity_state()` and `wp_interactivity_config()`.

### Asset Loading

- **Editor scripts** — Loaded only in the block editor
- **View scripts** — Loaded only on the frontend when the block is present
- **Styles** — Conditionally loaded per block

### Conditional Registration

Blocks can be conditionally disabled via the Aegis admin settings:

```php
if ( class_exists( '\Aegis\Admin\ConditionalLogicSettings' ) &&
    ! \Aegis\Admin\ConditionalLogicSettings::is_block_enabled( 'slider' ) ) {
    return;
}
```

### Duplicate Registration Guard

The framework includes guards to prevent duplicate registration when the Pro plugin's block package also registers the same blocks:

```php
$registry = WP_Block_Type_Registry::get_instance();

if ( ! $registry->is_registered( 'aegis/slider' ) ) {
    register_block_type( $slider_path );
}
```
