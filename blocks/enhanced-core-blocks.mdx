## Default Block Styles

Aegis applies consistent default styles to core blocks via `theme.json`. Key defaults include:

### Borders & Radius

Most blocks receive the global border radius token:

```css
border-radius: var(--wp--custom--border--radius); /* 6px */
```

Applied to: Image, Code, Column, Columns, HTML, List, Video, Audio, and more.

### Code Block

```
Font: JetBrains Mono (14px, medium weight)
Background: neutral-100
Padding: block gap on all sides
Border: none, with border radius
```

### Navigation

```
Font: Lexend (14px, medium weight)
```

### Buttons

```
Text: primary-25
Background: primary-500
Font: 14px, semi-bold, 20px line height
Padding: 10px 16px
Border radius: global token
```

### Links

```
Color: inherit
Decoration: underline
Hover/Focus: primary-500
```

### Separators

```
Color: neutral-200
Border: currentColor
```

## How Enhancements Work

Aegis block enhancements are implemented through the framework's `CoreBlocks` and `DesignSystem` namespaces:

- **PHP filters** — `register_block_type_args` modifies block registration to add attributes and supports.
- **Render filters** — `render_block_{name}` modifies block HTML output.
- **Editor assets** — Additional controls are injected into the block sidebar via editor scripts.
- **Conditional CSS** — Stylesheets are loaded only when a block is present on the page.

## Customization

Block styles can be overridden via:

1. **Site Editor** — Appearance → Editor → Styles → Blocks
2. **theme.json** — Override in `styles.blocks` section of a child theme
3. **CSS** — Target block classes directly (e.g., `.wp-block-code`)
