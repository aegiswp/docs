## Mini Cart

`User` `Designer` `Free`

Aegis includes a **Mini Cart** template part (`parts/mini-cart.html`) that provides a slide-out cart sidebar:

- Accessible from the header
- Shows cart contents without page reload
- Registered as an `uncategorized` template part area

### Mini-Cart Style Variants

Three CSS-based variants are available via `className` on the `woocommerce/mini-cart` block:

| Variant | Class | Description |
|---------|-------|-------------|
| **Default** | _(none)_ | Standard drawer with balanced spacing |
| **Compact** | `is-style-compact` | 48px thumbnails, tighter spacing, 13px fonts, hidden product metadata, condensed footer |
| **Rich** | `is-style-rich` | 96px thumbnails, larger fonts, metadata pills, 2px borders, stacked footer buttons |

#### Usage

Add the `className` attribute to the mini-cart block in any header pattern:

```html
<!-- Default -->
<!-- wp:woocommerce/mini-cart {} /-->

<!-- Compact -->
<!-- wp:woocommerce/mini-cart {"className":"is-style-compact"} /-->

<!-- Rich -->
<!-- wp:woocommerce/mini-cart {"className":"is-style-rich"} /-->
```

All variants include dark mode overrides and reduced motion support.

## Styling

WooCommerce elements inherit Aegis design tokens:

### Buttons

```css
.woocommerce .button,
.woocommerce button.button {
    background: var(--wp--preset--color--primary-500);
    color: var(--wp--preset--color--primary-25);
    border-radius: var(--wp--custom--border--radius);
}
```

### Forms

```css
.woocommerce input,
.woocommerce select {
    border: 1px solid var(--wp--custom--border--color);
    border-radius: var(--wp--custom--border--radius);
}
```

### Product Cards

```css
.wc-block-grid__product {
    border-radius: var(--wp--custom--border--radius);
}
```

## Customization

### Product Grid Layout

Use the Query Loop block with product post type:

```html
<!-- wp:query {"query":{"postType":"product","perPage":12}} -->
  <!-- wp:post-template {"layout":{"type":"grid","columnCount":4}} -->
    <!-- wp:post-featured-image /-->
    <!-- wp:post-title /-->
    <!-- wp:woocommerce/product-price /-->
    <!-- wp:woocommerce/product-button /-->
  <!-- /wp:post-template -->
<!-- /wp:query -->
```

### Shop Page

Customize the shop page template:

1. Go to **Appearance → Editor → Templates**
2. Select **Archive: Product**
3. Edit the layout with blocks

### Single Product

Customize product pages:

1. Edit the **Single: Product** template
2. Arrange WooCommerce blocks as needed
3. Add custom content sections

## Troubleshooting

### Styles not applying

1. Clear any caching plugins
2. Check for CSS conflicts
3. Ensure WooCommerce is updated

### Template issues

1. Verify template files exist
2. Check template hierarchy
3. Reset customizations if needed

## Multi-Step Checkout

`User` `Free`

The multi-step checkout template (`page-checkout-multi-step.html`) provides a step-by-step checkout flow:

- **Step-by-step flow** — Breaks checkout into discrete steps (contact, shipping, payment)
- **Progress indicator** — Visual step tracker showing current position
- **Reduced cognitive load** — Only one section visible at a time
- **Mobile-optimized** — Responsive layout with touch-friendly navigation
- **Localized JS** — Step labels and navigation text are translatable

### Usage

1. Go to **Appearance → Editor → Templates**
2. Assign the **Page: Checkout Multi-Step** template to your checkout page
3. The multi-step flow activates automatically

### Implementation

`Dev`

- **Class:** `Aegis\Checkout\MultiStep` (`src/Checkout/MultiStep.php`)
- **CSS:** `src/Checkout/css/multi-step.css` — conditionally loaded when the template is detected
- **JS:** `src/Checkout/js/multi-step.js` — step navigation and validation logic

## Framework Implementation

The WooCommerce integration is handled by:

- `Integrations\WooCommerce` — Main integration class
- Conditional loading — Integration CSS/JS loads only when WooCommerce is active
